HTML Status Codes

  • Name: Bandwidth Limit Exceeded
    Code: 509
    Purpose: This status code is used by many servers to indicate that the bandwidth limit imposed by the server administrator has been exceeded.
    Situation: If a website exceeds its allocated bandwidth, it might return a 509 status code to visitors until the limit is reset or increased.
  • Name: No Response
    Code: 444
    Purpose: This status code is used by the Nginx web server to indicate that the server has closed the connection without sending any response to the client.
    Situation: Nginx might return a 444 status code to drop a malicious request without sending any response, often used in firewall configurations.
  • Name: SSL Certificate Error
    Code: 495
    Purpose: This status code is used by Nginx to indicate that an SSL certificate error has occurred.
    Situation: If a client attempts to connect to a server using HTTPS with an invalid or untrusted SSL certificate, the server might return a 495 status code.
  • Name: SSL Certificate Required
    Code: 496
    Purpose: This status code is used by Nginx to indicate that a client certificate is required to establish an SSL connection.
    Situation: When a server requires a valid client certificate for authentication and the client does not provide one, the server might return a 496 status code.
  • Name: Network Read Timeout Error
    Code: 598
    Purpose: This status code is used by some proxy servers to indicate that a network read timeout has occurred.
    Situation: If a proxy server times out while reading from an upstream server, it might return a 598 status code to the client to indicate the timeout.
